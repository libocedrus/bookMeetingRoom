# iPhone 快捷指令设置

本文档用于配置 iPhone 自动判断是否需要提醒预定会议室。工作日判断和提醒判断由 GitHub Actions 预先生成静态 JSON，iPhone 快捷指令按当天日期读取对应结果，并根据结果开关闹钟。

## 固定配置

在“时钟”App 中提前创建两个闹钟，名称必须完全一致。

```text
07:48 【运行失败】请自查预定会议室
07:50 预定会议室
```

两个闹钟的用途：

```text
【运行失败】请自查预定会议室：快捷指令失败兜底闹钟
预定会议室：正常会议室预定提醒闹钟
```

GitHub Pages 每日数据地址格式：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/YYYY-MM-DD.json
```

快捷指令运行时会用 iPhone 当前日期拼出当天地址，例如：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/2026-05-12.json
```

## 快捷指令名称

创建一个快捷指令：

```text
会议室预定提醒
```

个人自动化只运行这个快捷指令。

## 快捷指令逻辑

```text
先打开 07:48 失败兜底闹钟
获取 iPhone 当前日期
格式化为 yyyy-MM-dd
拼出当天 daily JSON 地址
读取当天 JSON
检查 JSON 的 date 是否等于今天
检查 status 是否为 ok
根据 actionPolicy 开关 07:50 正常提醒闹钟
成功完成后关闭 07:48 失败兜底闹钟
```

如果网络失败、GitHub 返回异常、JSON 解析失败、获取字典值失败、`date` 不是今天，或 `status` 不是 `ok`，快捷指令不会关闭 07:48 失败兜底闹钟，所以 07:48 会响铃提醒你手动检查。

每日 JSON 中也包含 `shouldRemind`，但快捷指令使用 `actionPolicy` 判断。`actionPolicy` 是稳定字符串，不会被 iOS 本地化。

```text
open_alarm：需要提醒
close_alarm：无需提醒
```

## 快捷指令步骤

### 0. 【顶部输入设置】

保持默认即可。

```text
从 无处 接收 App 和其他 18 个
如果没有输入：继续
```

如果快捷指令顶部没有出现这块输入设置，不需要额外添加。

### 1. 【打开闹钟】

```text
闹钟：【运行失败】请自查预定会议室
操作：打开
运行时显示：关闭
```

这一步必须放在最前面。

### 2. 【当前日期】

```text
当前日期
```

### 3. 【格式化日期】

```text
日期：当前日期
日期格式：自定
格式字符串：yyyy-MM-dd
区域：默认
```

这一步生成当天日期文本，例如：

```text
2026-05-12
```

### 4. 【文本】

```text
内容：https://libocedrus.github.io/bookMeetingRoom/data/daily/格式化后的日期.json
```

实际内容由固定前缀、格式化后的日期变量、`.json` 三部分组成：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/
格式化后的日期
.json
```

最终效果示例：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/2026-05-12.json
```

### 5. 【获取 URL 内容】

```text
URL：第 4 步的文本
方法：GET
头部：不设置
```

后续所有“获取字典值”动作都从这个 `URL 的内容` 中读取。

### 6. 【获取字典值】

```text
在 URL 的内容 中获取 date 的值
```

### 7. 【文本】

```text
内容：上一步输出的字典值
```

这一步得到 JSON 中的 `date` 文本。

### 8. 【如果】

```text
条件：如果 文本 是 格式化后的日期
```

分支结构：

```text
如果分支：读取到的是今天的数据
否则分支：读取到的不是今天的数据
```

### 8.1 【获取字典值】

位置：第 8 步的“如果分支”里面。

```text
在 URL 的内容 中获取 status 的值
```

### 8.2 【文本】

位置：第 8 步的“如果分支”里面，紧跟在 8.1 后面。

```text
内容：上一步输出的字典值
```

### 8.3 【如果】

位置：第 8 步的“如果分支”里面，紧跟在 8.2 后面。

```text
条件：如果 文本 是 ok
```

分支结构：

```text
如果分支：数据状态正常
否则分支：数据状态异常
```

### 8.3.1 【获取字典值】

位置：第 8.3 步的“如果分支”里面。

```text
在 URL 的内容 中获取 actionPolicy 的值
```

### 8.3.2 【文本】

位置：第 8.3 步的“如果分支”里面，紧跟在 8.3.1 后面。

```text
内容：上一步输出的字典值
```

### 8.3.3 【如果】

位置：第 8.3 步的“如果分支”里面，紧跟在 8.3.2 后面。

```text
条件：如果 文本 是 open_alarm
```

分支结构：

```text
如果分支：需要提醒
否则分支：无需提醒
```

#### 8.3.3.1 【打开闹钟】

位置：第 8.3.3 步的“如果分支”里面。

```text
闹钟：预定会议室
操作：打开
运行时显示：关闭
```

#### 8.3.3.2 【关闭闹钟】

位置：第 8.3.3 步的“如果分支”里面，紧跟在 8.3.3.1 后面。

```text
闹钟：【运行失败】请自查预定会议室
操作：关闭
运行时显示：关闭
```

#### 8.3.3.3 【显示通知】

位置：第 8.3.3 步的“如果分支”里面，紧跟在 8.3.3.2 后面。

```text
通知内容：预约3个工作日后的会议室
```

#### 8.3.3.4 【关闭闹钟】

位置：第 8.3.3 步的“否则分支”里面。

```text
闹钟：预定会议室
操作：关闭
运行时显示：关闭
```

#### 8.3.3.5 【关闭闹钟】

位置：第 8.3.3 步的“否则分支”里面，紧跟在 8.3.3.4 后面。

```text
闹钟：【运行失败】请自查预定会议室
操作：关闭
运行时显示：关闭
```

#### 8.3.3.6 【显示通知】

位置：第 8.3.3 步的“否则分支”里面，紧跟在 8.3.3.5 后面。

```text
通知内容：检查完毕，无需预约
```

### 8.3.4 【结束如果】

这是第 8.3.3 步 `actionPolicy` 判断的结束位置。

### 8.3.5 【获取字典值】

位置：第 8.3 步的“否则分支”里面。

```text
在 URL 的内容 中获取 message 的值
```

### 8.3.6 【显示通知】

位置：第 8.3 步的“否则分支”里面，紧跟在 8.3.5 后面。

```text
通知内容：上一步输出的 message 字典值
```

这个分支不关闭失败兜底闹钟。

### 8.4 【结束如果】

这是第 8.3 步 `status` 判断的结束位置。

### 8.5 【结束如果】

这是第 8 步 `date` 判断的结束位置。

## 关键层级要求

```text
8.1 到 8.4 必须全部放在第 8 步 date = 今天的“如果分支”里面
8.3.1 到 8.3.4 必须全部放在第 8.3 步 status = ok 的“如果分支”里面
8.3.3.1 到 8.3.3.3 必须放在第 8.3.3 步 actionPolicy = open_alarm 的“如果分支”里面
8.3.3.4 到 8.3.3.6 必须放在第 8.3.3 步 actionPolicy = open_alarm 的“否则分支”里面
8.3.5 和 8.3.6 必须放在第 8.3 步 status = ok 的“否则分支”里面
读取到的 date 不是今天时，不关闭失败兜底闹钟
status 不是 ok 时，不关闭失败兜底闹钟
```

## 关闭闹钟动作的系统弹窗

快捷指令中每一个“打开闹钟”或“关闭闹钟”动作，都要关闭动作自带的：

```text
运行时显示
```

否则 iOS 会弹出类似下面的系统结果面板：

```text
07:48 的闹钟现已 On.
```

这是“闹钟”动作自己的运行结果展示，不是“显示通知”。关闭“运行时显示”后，快捷指令仍会正常开关闹钟，但不会弹出这个结果面板。

## 创建自动化

在“快捷指令”App 中创建个人自动化：

```text
触发条件：每天固定时间
时间：07:45 或 07:46
动作：运行快捷指令“会议室预定提醒”
运行前询问：关闭
```

自动化时间必须早于 07:48。推荐：

```text
07:46 运行快捷指令
07:48 失败兜底闹钟
07:50 正常提醒闹钟
```

## 运行结果

完整执行成功且需要提醒：

```text
date = 今天
status = ok
actionPolicy = open_alarm
打开“预定会议室”
关闭“【运行失败】请自查预定会议室”
显示通知“预约3个工作日后的会议室”
```

完整执行成功且无需提醒：

```text
date = 今天
status = ok
actionPolicy = close_alarm
关闭“预定会议室”
关闭“【运行失败】请自查预定会议室”
显示通知“检查完毕，无需预约”
```

执行失败、date 不是今天，或 status 不是 ok：

```text
“【运行失败】请自查预定会议室”保持打开
07:48 响铃
```
