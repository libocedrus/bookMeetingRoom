# iPhone 快捷指令设置

本文档用于配置 iPhone 自动开关“预定会议室”闹钟。工作日判断和提醒判断已经由 GitHub Actions 生成静态 JSON，iPhone 快捷指令只负责读取结果并开关闹钟。

## 固定名称

下面 3 个名称必须完全一致，包括大小写、中文、连字符和空格。不要随意改名：

```text
闹钟名称：预定会议室
核心快捷指令：检查会议室预定提醒
访问检查快捷指令：检查会议室预定提醒-访问检查
```

原因：第二个快捷指令中的 `x-callback-url` 已经把快捷指令名称做了 URL 转码。如果名称不一致，iOS 就找不到对应快捷指令，异常回调也会失效。

GitHub Pages 数据地址为：

```text
https://libocedrus.github.io/bookMeetingRoom/data/today.json
```

## 创建闹钟

在“时钟”App 中创建一个每天固定时间的闹钟：

```text
名称：预定会议室
建议时间：09:00
```

## 快捷指令一：检查会议室预定提醒

这是核心快捷指令，负责读取 `today.json` 中的 `shouldRemind`，并开关闹钟。

### 1. 获取 URL 内容

添加动作：

```text
获取 URL 内容
```

配置：

```text
URL：https://libocedrus.github.io/bookMeetingRoom/data/today.json
方法：GET
```

### 2. 获取 shouldRemind

添加动作：

```text
获取字典值
```

配置：

```text
在 URL 的内容 中获取 shouldRemind 的值
```

添加动作：

```text
文本
```

文本内容选择上一步输出的 `shouldRemind` 或 `字典值`。

### 3. 判断 shouldRemind

添加动作：

```text
如果
```

条件：

```text
如果 文本 是 true
```

在“如果”分支中添加：

```text
打开闹钟“预定会议室”
显示通知“预约3个工作日后的会议室”
```

在“否则”分支中添加：

```text
关闭闹钟“预定会议室”
```

最终结构：

```text
获取 URL 内容
获取 shouldRemind
文本 shouldRemind

如果 shouldRemind 是 true
  打开闹钟“预定会议室”
  显示通知“预约3个工作日后的会议室”
否则
  关闭闹钟“预定会议室”
结束如果
```

## 快捷指令二：检查会议室预定提醒-访问检查

这是自动化实际运行的快捷指令。它负责：

```text
正常情况：用 x-callback-url 运行“检查会议室预定提醒”
异常情况：如果核心快捷指令报错，x-error 会重新运行本快捷指令，并传入 error
```

### 1. 保持顶部输入设置默认

当你在这个快捷指令中选择“输入快捷指令的信息”后，顶部会出现类似：

```text
从 无处 接收 App 和其他 18 个
如果没有输入：继续
```

这里不用改，保持默认即可。

含义是：

```text
正常自动化运行时：没有输入，但继续执行
x-error 回调运行时：会收到 error 文本，所以“输入快捷指令的信息”有值
```

### 2. 判断是否为异常回调

添加动作：

```text
如果
```

点蓝色变量位置，选择：

```text
输入快捷指令的信息
```

条件设置为：

```text
有任何值
```

最终应显示为：

```text
如果 输入快捷指令的信息 有任何值
```

在“如果”分支中添加：

```text
显示通知“⚠️ 无法读取会议室提醒数据，请手动确认是否需要预约会议室”
打开闹钟“预定会议室”
停止此快捷指令
```

“否则”分支留空即可。

### 3. 添加 URL

在整个“如果”动作结束之后，也就是 `结束如果` 后面，添加动作：

```text
URL
```

URL 内容填写下面这一整段：

```text
shortcuts://x-callback-url/run-shortcut?name=%E6%A3%80%E6%9F%A5%E4%BC%9A%E8%AE%AE%E5%AE%A4%E9%A2%84%E5%AE%9A%E6%8F%90%E9%86%92&x-error=shortcuts%3A%2F%2Frun-shortcut%3Fname%3D%25E6%25A3%2580%25E6%259F%25A5%25E4%25BC%259A%25E8%25AE%25AE%25E5%25AE%25A4%25E9%25A2%2584%25E5%25AE%259A%25E6%258F%2590%25E9%2586%2592-%25E8%25AE%25BF%25E9%2597%25AE%25E6%25A3%2580%25E6%259F%25A5%26input%3Dtext%26text%3Derror
```

这条 URL 的含义是：

```text
运行：检查会议室预定提醒
如果运行失败：重新运行 检查会议室预定提醒-访问检查，并传入 error 文本
```

### 4. 打开 URL

继续添加动作：

```text
打开 URL
```

打开上一步的 URL。

最终结构：

```text
如果 输入快捷指令的信息 有任何值
  显示通知“⚠️ 无法读取会议室提醒数据，请手动确认是否需要预约会议室”
  打开闹钟“预定会议室”
  停止此快捷指令
结束如果

URL shortcuts://x-callback-url/run-shortcut?name=...
打开 URL
```

## 创建自动化

在“快捷指令”App 中创建个人自动化：

```text
触发条件：每天固定时间
时间：比“预定会议室”闹钟早 5 到 10 分钟
动作：运行快捷指令“检查会议室预定提醒-访问检查”
运行前询问：关闭
```

推荐：

```text
08:50 自动判断
09:00 预定会议室闹钟
```

注意：自动化运行的是 `检查会议室预定提醒-访问检查`，不是 `检查会议室预定提醒`。

## 异常处理

如果核心快捷指令出现以下问题：

```text
GitHub URL 访问失败
网络超时
返回内容不是 JSON
获取字典值失败
其他运行错误
```

`x-error` 会让 `检查会议室预定提醒-访问检查` 再运行一次，并带上 `error` 输入。此时它会进入异常分支：

```text
显示通知“⚠️ 无法读取会议室提醒数据，请手动确认是否需要预约会议室”
打开闹钟“预定会议室”
停止此快捷指令
```

## 调试

第一次配置后，先手动运行：

```text
检查会议室预定提醒-访问检查
```

如果能正常调用 `检查会议室预定提醒`，说明 GitHub Pages 可以访问，且核心快捷指令运行正常。

如果你看到异常通知，并且 `预定会议室` 闹钟被打开，说明核心快捷指令访问 GitHub 或解析 JSON 时失败。

如果改了任意快捷指令名称，需要重新生成 `x-callback-url` 中的转码内容；否则不要改名。
