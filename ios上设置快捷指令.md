# iPhone 快捷指令设置

本文档用于配置 iPhone 自动判断是否需要提醒预定会议室。工作日判断和提醒判断由 GitHub Actions 预先生成静态 JSON，iPhone 快捷指令只负责读取当天结果并开关闹钟。

## 固定配置

需要在“时钟”App 中提前创建两个闹钟，名称必须完全一致。

```text
07:48 【运行失败】请自查预定会议室
07:50 预定会议室
```

两个闹钟的用途：

```text
【运行失败】请自查预定会议室：快捷指令失败兜底闹钟
预定会议室：正常会议室预定提醒闹钟
```

GitHub Pages 每日数据地址格式：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/YYYY-MM-DD.json
```

例如：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/2026-05-12.json
```

`today.json` 仅作为调试和兼容入口，正式快捷指令不读取 `today.json`。

## 快捷指令名称

创建一个快捷指令：

```text
会议室预定提醒
```

个人自动化只运行这个快捷指令。

## 快捷指令逻辑

快捷指令采用兜底优先逻辑：

```text
先打开 07:48 失败兜底闹钟
再把 iPhone 当前日期格式化为 yyyy-MM-dd
用当天日期拼出 /data/daily/YYYY-MM-DD.json
读取当天 JSON
检查 JSON 里的 date 是否等于今天
检查 status 是否为 ok
根据 actionPolicy 开关 07:50 正常提醒闹钟
完整执行成功后，关闭 07:48 失败兜底闹钟
```

如果网络失败、GitHub 返回异常、JSON 解析失败、获取字典值失败、`date` 不是今天，或 `status` 不是 `ok`，快捷指令不会关闭 07:48 失败兜底闹钟，所以仍然会响铃提醒你手动检查。

注意：每日 JSON 里也有 `shouldRemind`，但它是布尔值。iOS 快捷指令把布尔值转成文本时，会按系统语言显示为 `是` 或 `否`，而不是 `true` 或 `false`。因此快捷指令中使用字符串字段 `actionPolicy` 做判断，更稳定。

## 快捷指令步骤

### 0. 【顶部输入设置】

保持默认即可。这个快捷指令由个人自动化直接运行，不需要接收外部输入。

```text
从 无处 接收 App 和其他 18 个
如果没有输入：继续
```

如果你的快捷指令顶部没有出现这块输入设置，也不用额外添加。

### 1. 【打开闹钟】

```text
闹钟：【运行失败】请自查预定会议室
操作：打开
运行时显示：关闭
```

这一步必须放在最前面。它是失败兜底：只要后续访问 GitHub、解析 JSON 或状态判断失败，这个闹钟就会保持打开，并在 07:48 响铃。

### 2. 【当前日期】

```text
当前日期
```

这一步得到 iPhone 当前日期。

### 3. 【格式化日期】

```text
日期：当前日期
日期格式：自定
格式字符串：yyyy-MM-dd
```

这一步得到今天的文本日期，例如：

```text
2026-05-12
```

### 4. 【文本】

```text
内容：https://libocedrus.github.io/bookMeetingRoom/data/daily/格式化后的日期.json
```

实际填写时，把第 3 步的“格式化后的日期”变量插入到 URL 中间。最终文本应类似：

```text
https://libocedrus.github.io/bookMeetingRoom/data/daily/2026-05-12.json
```

这一步用于拼出当天 JSON 地址。

### 5. 【获取 URL 内容】

```text
URL：第 4 步的文本
方法：GET
```

后续所有“获取字典值”动作，都从这个 `URL 的内容` 中读取。

### 6. 【获取字典值】

```text
在 URL 的内容 中获取 date 的值
```

这一步得到每日 JSON 里的数据日期。

### 7. 【文本】

```text
内容：上一步输出的 date 或 字典值
```

这一步用于把 JSON 里的 `date` 转成可比较的文本。

### 8. 【如果】

```text
条件：如果 date 文本 是 格式化后的日期
```

分支结构：

```text
如果分支：读取到的是今天的数据，继续读取 status
否则分支：读取到的不是今天的数据，停止此快捷指令
```

在“否则”分支中添加：

```text
停止此快捷指令
```

注意：如果每日 JSON 的 `date` 和 iPhone 今天日期不一致，必须停止快捷指令，不要继续读取 `status` 或 `actionPolicy`。

### 8.1 【获取字典值】

位置：放在第 8 步 `date = 今天` 的“如果分支”里面。

```text
在 URL 的内容 中获取 status 的值
```

### 8.2 【文本】

位置：放在第 8 步 `date = 今天` 的“如果分支”里面，紧跟在 8.1 后面。

```text
内容：上一步输出的 status 或 字典值
```

### 8.3 【如果】

位置：放在第 8 步 `date = 今天` 的“如果分支”里面，紧跟在 8.2 后面。

```text
条件：如果 status 文本 是 ok
```

分支结构：

```text
如果分支：status 正常，继续读取 actionPolicy
否则分支：status 异常，停止此快捷指令
```

在“否则”分支中添加：

```text
停止此快捷指令
```

注意：`status` 不是 `ok` 时，不要关闭失败兜底闹钟，也不要显示通知。让 07:48 的失败兜底闹钟响铃即可。

### 8.3.1 【获取字典值】

位置：放在第 8.3 步 `status = ok` 的“如果分支”里面。

```text
在 URL 的内容 中获取 actionPolicy 的值
```

这一步得到的是字符串，不会被 iOS 翻译。

### 8.3.2 【文本】

位置：放在第 8.3 步 `status = ok` 的“如果分支”里面，紧跟在 8.3.1 后面。

```text
内容：上一步输出的 actionPolicy 或 字典值
```

### 8.3.3 【如果】

位置：放在第 8.3 步 `status = ok` 的“如果分支”里面，紧跟在 8.3.2 后面。

```text
条件：如果 actionPolicy 文本 是 open_alarm
```

分支结构：

```text
如果分支：需要提醒，打开“预定会议室”闹钟
否则分支：无需提醒，关闭“预定会议室”闹钟
```

#### 8.3.3.1 【打开闹钟】

位置：放在第 8.3.3 步 `actionPolicy = open_alarm` 的“如果分支”里面。

```text
闹钟：预定会议室
操作：打开
运行时显示：关闭
```

#### 8.3.3.2 【显示通知】

位置：放在第 8.3.3 步 `actionPolicy = open_alarm` 的“如果分支”里面，紧跟在 8.3.3.1 后面。

```text
通知内容：预约3个工作日后的会议室
```

这是整个快捷指令中唯一需要主动显示的通知。只有打开 `预定会议室` 闹钟后才显示。

#### 8.3.3.3 【关闭闹钟】

位置：放在第 8.3.3 步 `actionPolicy = open_alarm` 的“否则分支”里面。

```text
闹钟：预定会议室
操作：关闭
运行时显示：关闭
```

这个分支不显示通知。

### 8.3.4 【结束如果】

这是第 8.3.3 步 `actionPolicy` 判断的结束位置。

### 8.3.5 【关闭闹钟】

位置：放在第 8.3 步 `status = ok` 的“如果分支”里面，并且必须放在第 8.3.4 步后面。

```text
闹钟：【运行失败】请自查预定会议室
操作：关闭
运行时显示：关闭
```

只有 `date = 今天`、`status = ok` 且 `actionPolicy` 分支完整执行后，才能关闭失败兜底闹钟。

### 8.3.6 【停止此快捷指令】

位置：放在第 8.3 步 `status = ok` 的“否则分支”里面。

```text
停止此快捷指令
```

这个分支代表 `status` 不是 `ok`。停止后不要再添加关闭失败兜底闹钟的动作，让 07:48 的失败兜底闹钟保持打开。

### 8.4 【结束如果】

这是第 8.3 步 `status` 判断的结束位置。

### 8.5 【停止此快捷指令】

位置：放在第 8 步 `date = 今天` 的“否则分支”里面。

```text
停止此快捷指令
```

这个分支代表读取到的每日 JSON 不是今天的数据。停止后不要关闭失败兜底闹钟。

### 9. 【结束如果】

这是第 8 步 `date` 判断的结束位置。

```text
   如果没有输入：继续

1. 【打开闹钟】
   闹钟：【运行失败】请自查预定会议室
   操作：打开
   运行时显示：关闭

2. 【当前日期】

3. 【格式化日期】
   日期：当前日期
   日期格式：自定
   格式字符串：yyyy-MM-dd

4. 【文本】
   内容：https://libocedrus.github.io/bookMeetingRoom/data/daily/格式化后的日期.json

5. 【获取 URL 内容】
   URL：第 4 步的文本
   方法：GET

6. 【获取字典值】
   在 URL 的内容 中获取 date 的值

7. 【文本】
   内容：date

8. 【如果】
   条件：如果 date 文本 是 格式化后的日期
   ├─ 如果分支：读取到的是今天的数据
   │  8.1 【获取字典值】
   │      在 URL 的内容 中获取 status 的值
   │

   │      条件：如果 status 文本 是 ok
   │      ├─ 如果分支：status 正常
   │      │  8.3.1 【获取字典值】
   │      │      在 URL 的内容 中获取 actionPolicy 的值
   │      │
   │      │  8.3.2 【文本】
   │      │      内容：actionPolicy
   │      │
   │      │  8.3.3 【如果】
   │      │      条件：如果 actionPolicy 文本 是 open_alarm
   │      │      ├─ 如果分支：需要提醒
   │      │      │  8.3.3.1 【打开闹钟】
   │      │      │          闹钟：预定会议室
   │      │      │          操作：打开
   │      │      │          运行时显示：关闭
   │      │      │
   │      │      │  8.3.3.2 【显示通知】
   │      │      │          通知内容：预约3个工作日后的会议室
   │      │      │
   │      │      └─ 否则分支：无需提醒
   │      │         8.3.3.3 【关闭闹钟】
   │      │                 闹钟：预定会议室
   │      │                 操作：关闭
   │      │                 运行时显示：关闭
   │      │
   │      │  8.3.4 【结束如果】
   │      │
   │      │  8.3.5 【关闭闹钟】
   │      │      闹钟：【运行失败】请自查预定会议室
   │      │      操作：关闭
   │      │      运行时显示：关闭
   │      │
   │      └─ 否则分支：status 异常
   │         8.3.6 【停止此快捷指令】
   │
   │  8.4 【结束如果】
   │
   └─ 否则分支：读取到的不是今天的数据
      8.5 【停止此快捷指令】

9. 【结束如果】
```

关键层级要求：

```text
8.1 到 8.4 必须全部放在第 8 步 date = 今天的“如果分支”里面
8.3.1 到 8.3.5 必须全部放在第 8.3 步 status = ok 的“如果分支”里面
8.3.3.1 和 8.3.3.2 必须放在第 8.3.3 步 actionPolicy = open_alarm 的“如果分支”里面
8.3.3.3 必须放在第 8.3.3 步 actionPolicy = open_alarm 的“否则分支”里面
8.3.5 必须放在第 8.3.4 之后，也就是 actionPolicy 判断完整结束之后
date 不是今天时，只停止快捷指令，不关闭失败兜底闹钟
status 不是 ok 时，只停止快捷指令，不关闭失败兜底闹钟
显示通知只放在 8.3.3.1 打开“预定会议室”之后
```

## 关闭闹钟动作的系统弹窗

快捷指令中每一个“打开闹钟”或“关闭闹钟”动作，都要关闭动作自带的：

```text
运行时显示
```

否则 iOS 会弹出类似下面的系统结果面板：

```text
07:48 的闹钟现已 On.
```

这是“闹钟”动作自己的运行结果展示，不是本快捷指令中的“显示通知”。关闭“运行时显示”后，快捷指令仍会正常开关闹钟，但不会弹出这个结果面板。

## 创建自动化

在“快捷指令”App 中创建个人自动化：

```text
触发条件：每天固定时间
时间：07:45 或 07:46
动作：运行快捷指令“会议室预定提醒”
运行前询问：关闭
```

自动化时间必须早于 07:48。推荐：

```text
07:46 运行快捷指令
07:48 失败兜底闹钟
07:50 正常提醒闹钟
```

## 运行结果

完整执行成功且需要提醒：

```text
date = 今天
status = ok
actionPolicy = open_alarm
打开“预定会议室”
显示通知“预约3个工作日后的会议室”
关闭“【运行失败】请自查预定会议室”
```

完整执行成功且无需提醒：

```text
date = 今天
status = ok
actionPolicy = close_alarm
关闭“预定会议室”
关闭“【运行失败】请自查预定会议室”
不显示通知
```

执行失败、date 不是今天，或 status 不是 ok：

```text
“【运行失败】请自查预定会议室”保持打开
07:48 响铃
不依赖通知提醒
```
